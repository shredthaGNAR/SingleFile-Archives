    Being able to merge entries after creating would be awesome.

    But going right along with this, and even more important really, is preventing the duplicates in the first place.

    Shouldn't ProtonPass Chromium/Chrome extension be more intelligent about heading off duplicate entries right when it sees me wanting to sign into a website for which it has no Valult entry with a matching WEBSITES field?

    IOW, ideally, the behavior I would like to see is:

    I go to a website where I already have an account.

    a) Plan A: ProtonPass tries to find a Vault entry with a matching WEBSITES field. (It already does this.) If it cannot find a matching WEBSITES field, then it should fall back on Plan B:

    b) Plan B: It should try to intelligently "guess", which entry I might already have in my ProtonPass Vault, but I just don't have the Website/URL entered, yet, based upon grepping for similar text in any field of all entries.

    For instance, if I have an entry with TITLE "example.com email" but nothing in the WEBSITES field, then when I go to example.com/sign-in, ProtonPass extension should "see" the TITLE and see that it has many of the same sequences of characters, and and ask me: "Is this the same as the site you're on now? And if I say yes, then just add the website/url to the WEBSITES field of the previously existing entry.

    And if it CANNOT find any reasonable entry, then..

    c) Plan C: ProtonPass extension should ask me: "I can't find this website in my vaults by URL or name. But would you like to search for and add this sign-in URL to an already existing Vault entry?" If I respond, "Yes", then it should let me pick one manually from among the already-existing entries in the vault, and then it would add the website/url to the WEBSITES field of that entry. If I respond, "No", then of course it could just go ahead and ask me (as it already does), "Shall I make a new entry?".
