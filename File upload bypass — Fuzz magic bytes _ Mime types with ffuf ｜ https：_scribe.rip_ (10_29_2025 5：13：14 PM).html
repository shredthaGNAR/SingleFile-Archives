File upload bypass — Fuzz magic bytes / Mime types with ffuf

Intro

It’s common to see custom web apps with file upload functionality use unsafe checks on file extensions, content types, file magic headers / mime types, …etc to filter web shells.

Since there are already plenty of online guides and wordlists for fuzzing file extension and content type, I will discuss how to fuzz magic bytes for PHP file upload bypass.

Scenario

An internal PHP web app for file hosting allows only files with PE magic bytes to be uploaded

Vulernable source code

  1. It will move the uploaded file from a temporary directory to `/var/www/html/upload/file.tmp then check if it has PE file’s magic bytes.
  2. If true, the file will be renamed back to its original name.
  • The check can be bypassed by spoofing magic bytes to 4D5A*

upload.php

<?php
/* Get the name of the uploaded file */
$filename = $_FILES['file']['name'];

/* Choose where to save the uploaded file */
$tmp_location = "upload/file.tmp";
$location = "upload/".$filename;

/* Move the file temporary */
move_uploaded_file($_FILES['file']['tmp_name'], $tmp_location);

/* Check MagicBytes MZ PEFILE 4D5A*/
$F=fopen($tmp_location,"r");
$magic=fread($F,2);
fclose($F);
$magicbytes = strtoupper(substr(bin2hex($magic),0,4));
error_log(print_r("Magicbytes:" . $magicbytes, TRUE));

/* if its not a PEFILE block it - str_contains onlz php 8*/
if ( strpos($magicbytes, '4D5A') === false ) {
        echo "Unsupported file typ!\n";
        exit ();
}

rename($tmp_location, $location);

?>
